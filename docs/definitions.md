# Definitions — Precise Metric Definitions Used in This Analysis

Ambiguous definitions are a core weakness of Summer.fi's article. This file pins down
exactly what each metric means in our analysis.

## TVL (Total Value Locked)

**Our definition:** Sum of `totalAssets()` across all vaults included in a given scope,
converted to USD at the pricing source specified below.

**Scope matters:**
- **Lazy Summer TVL:** Only vaults in the Lazy Summer Protocol (yield aggregation vaults).
  DeFiLlama slug: `lazy-summer-protocol`.
- **Summer.fi TVL:** All Summer.fi deployments including Maker/Sky positions, borrow positions,
  and Lazy Summer vaults. DeFiLlama slug: `summer.fi`.
- The article conflates these two scopes. We always specify which.

**Strategy accounting:** If vaults deploy assets to external strategies, `totalAssets()`
should reflect the vault's total claim (including external positions). We verify this by
comparing `totalAssets()` to the vault's direct token balance — differences indicate strategy
deployment.

## Fees / Tips

**Definition:** Value extracted from vault depositors via share dilution ("tip streams").
Measured as the USD value of shares minted to fee-recipient addresses per unit time.

**Not fees:** Underlying yield earned by the vault and passed to depositors is *not* a protocol
fee — it's supply-side revenue.

**Effective fee rate:** `(annual fees USD) / (average AUM USD)`, expressed as a percentage.
- Per vault: computed individually
- Weighted average: `sum(vault fees) / sum(vault AUM)`, annualized

## Revenue

**Protocol revenue:** Total tips/fees collected by the protocol (all recipients).
**Staker revenue:** The portion of tips routed to the staker reward module, valued in USD.
- Valued at the block of deposit/claim using `convertToAssets()` for LV tokens.
**Treasury revenue:** The portion routed to the DAO treasury.

## Staker Share

**Definition:** `staker_revenue_usd / total_protocol_tips_usd` over a given period.
- Claimed: 20% (per DeFiLlama methodology and Summer.fi docs)
- We measure the realized ratio from on-chain flows.

## Emissions vs Revenue

**Emission rewards:** SUMR tokens distributed from the community allocation to stakers.
These are inflationary — they don't represent value generated by the protocol.
**Revenue rewards:** USDC-denominated LV vault tokens distributed from protocol tips.
These are non-inflationary — funded by actual fee revenue.
**Revenue ratio:** `revenue_rewards_usd / (revenue_rewards_usd + emission_rewards_usd)`

## Pricing Sources

| Asset | Primary | Secondary |
|-------|---------|-----------|
| USDC, USDT | $1.00 (unless depeg) | DeFiLlama coins API |
| ETH | Chainlink ETH/USD on Base | DeFiLlama coins API |
| SUMR | DeFiLlama coins API | DEX TWAP (Uniswap v3 on Base) |
| LV tokens | `convertToAssets()` at block | Manual NAV calculation |
